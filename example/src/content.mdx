# @diplodoc/mdx-extension Demo

This page demonstrates the capabilities of `@diplodoc/mdx-extension` for embedding MDX/JSX within markdown content.

<ThemeToggle/>

## Basic Usage

### 1. Simple JSX Component

```markdown
<Alert theme="warning" message="This is an alert component rendered via MDX"/>
```

Renders:

<Alert theme="warning" message="This is an alert component rendered via MDX"/>

### 2. With Props and Children

```markdown
<Button
  view="normal"
  onClick={() => alert('Clicked!')}
>
  Click Me
</Button>
```

Renders:

<Button view="normal" onClick={() => alert('Clicked!')}>Click Me</Button>

## MDX Syntax Variations

### 1. Explicit MDX Tags

```markdown
<MDX>
  <div style={{color: 'red', padding: '1rem'}}>
    This content is wrapped in explicit MDX tags
  </div>
</MDX>
```

Renders:

<MDX>
    <div style={{color: 'red', padding: '1rem'}}>This content is wrapped in explicit MDX tags</div>
</MDX>

### 2. JSX Fragment Shortcut

```markdown
<>
  <b>Fragment</b> content <i>works</i> too!
</>
```

Renders:

    <> <b>Fragment</b> content <i>works</i> too! </>

## Markdown Integration

MDX components work seamlessly with regular markdown:

```markdown
## Heading 2

This is **markdown** text with an embedded component:

<Label theme="success">Active</Label>

- List item 1
- List item 2
```

Renders:

## Heading 2

This is **markdown** text with an embedded component:

<Label theme="success">Active</Label>

- List item 1
- List item 2

        'SSR': ['✓', '✓', '✗'],

## Limitations

1. Components must be registered in the `components` object passed to the renderer
2. Complex state management should be handled in parent components
3. Some markdown formatting may need escaping in JSX blocks
